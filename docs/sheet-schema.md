# Схема листа

Источник анализа: `C:\Users\kiril\Desktop\АУКЦИОНЫ 2026.xlsx`.

Книга содержит листы:

- `2026 (44)`
- `2026 (223)`
- `2026 (КЗ)`

Для MVP используется только лист `2026 (44)`. В образце лист имеет 3621 строку и 21 колонку. Реальная строка заголовков — первая строка. Заголовки имеют серую заливку `FFCCCCCC`.

При чтении образца OpenPyXL предупредил, что несколько ячеек в колонке `ИНН заказчика` помечены как даты, хотя содержат числовые идентификаторы. Поэтому приложение не полагается на Excel-тип ячейки для закупочных номеров и ИНН: такие значения нормализуются как строки.

## Обязательные рабочие колонки

Приложение ищет колонки по заголовкам, а не по буквам. В качестве колонки номера закупки поддерживаются `Номер аукциона` и `Номер закупки`.

Минимальный набор для работы:

- `Номер аукциона` или `Номер закупки`
- `Дата окончания приёма заявок`
- `Объект закупки`
- `Статус`
- `Победная ставка`
- `Кто выиграл`
- `Поставляемый товар`

## Добавляемые служебные колонки

Если колонок нет, команда `python -m app.cli init-sheets` добавляет их справа:

- `ИНН победителя`
- `Текущая цена контракта`
- `Изменение контракта`
- `Ко вниманию`
- `Автостатус`
- `Ручная фиксация`
- `Дата последней проверки`
- `Дата последнего автообновления`
- `Источник протокола`
- `Источник контракта`
- `Источник спецификации`
- `Ссылка на реестровую запись`
- `Ошибка обработки`
- `Первоначальный победитель`

Колонки `Ко вниманию` и `Ручная фиксация` должны быть оформлены в Google Sheets как чекбоксы. В MVP они записываются как булевы значения; оформление чекбоксов оставлено в batchUpdate-слое.

## Пропускаемые строки

Приложение пропускает пустые строки, строки без номера закупки, повторные заголовки, разделители и технические пояснения. Перед каждой записью строка повторно ищется по номеру закупки, потому что сотрудники могут перемещать строки.
