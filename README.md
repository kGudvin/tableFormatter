# Автозаполнение результатов закупок 44-ФЗ

Сервис постоянно работает на VPS и заполняет результаты завершившихся закупок 44-ФЗ в постоянной Google Таблице. Веб-приложение принимает команды, а отдельный worker выполняет их из устойчивой очереди PostgreSQL. Поэтому закрытие страницы или перезапуск веб-контейнера не обрывает уже принятое задание.

## Архитектура

```text
app/
  config/      настройки из окружения
  domain/      бизнес-правила, ИНН, деньги, товары, состояния
  db/          SQLAlchemy-модели и сессии
  sheets/      Google Sheets API, поиск заголовков, служебные колонки
  sources/     интерфейс источников и адаптер ЕИС 44-ФЗ
  parsers/     извлечение текста из HTML/XML/PDF/DOCX/XLSX
  services/    обработчик закупок и сбор официального результата
  review/      синхронизация листа ручной проверки
  scheduler/   фоновый запуск
  api/         FastAPI: /health и подключение веб-оболочки
  web/         внутренняя панель администратора
  cli/         команды администратора
migrations/    Alembic-миграции PostgreSQL
docs/          бизнес-правила и схема листа
tests/         unit/integration тесты
```

PostgreSQL хранит рабочее состояние, аудит, снимки автоматически записанных полей и очередь ручной проверки. Google Таблица остаётся рабочим интерфейсом сотрудников.

Если в ЕИС для товара отсутствует товарный знак, но указан реестровый номер ПП РФ №719, сервис находит производителя в официальной выгрузке действующих записей ГИСП. Выгрузка индексируется локально и обновляется не чаще одного раза в сутки.

## Требования к VPS

- Ubuntu 24.04
- Docker и Docker Compose
- доступ исходящего HTTPS к Google Sheets API и публичным страницам ЕИС
- JSON-ключ сервисного аккаунта Google, не размещённый в Git

## Подготовка Google

1. В Google Cloud создайте проект и сервисный аккаунт.
2. Включите Google Sheets API.
3. Создайте JSON-ключ сервисного аккаунта.
4. Положите ключ на VPS, например `/opt/procurement/secrets/google-service-account.json`.
5. Откройте рабочую Google Таблицу и выдайте email сервисного аккаунта права редактора.
6. Возьмите `GOOGLE_SPREADSHEET_ID` из URL таблицы: часть между `/d/` и `/edit`.

## Настройка

```bash
cp .env.example .env
```

Заполните:

- `GOOGLE_APPLICATION_CREDENTIALS=/run/secrets/google-service-account.json`
- `GOOGLE_SERVICE_ACCOUNT_FILE=/opt/procurement/secrets/google-service-account.json`
- `GOOGLE_SPREADSHEET_ID=...`
- `GOOGLE_MAIN_SHEET=2026 (44)`
- `GOOGLE_REVIEW_SHEET=Требуется проверка`
- `WEB_UI_TOKEN=...` для защиты веб-оболочки заголовком или cookie `admin_token`

Секреты не коммитятся. JSON-ключ монтируется в контейнер только для чтения.

## Запуск

```bash
docker compose build
docker compose up -d
docker compose logs -f app
```

Миграции PostgreSQL выполняются автоматически командой `alembic upgrade head` при старте контейнера приложения.

## Веб-оболочка

Панель доступна внутри контейнерной сети по `/ui/` на порту `8080`. В ней есть:

- проверка базовой настройки;
- инициализация служебных колонок и листа проверки;
- инспекция схемы листа;
- запуск первичной обработки;
- запуск обработки очереди;
- синхронизация ручных проверок;
- диагностика одной закупки с опциональной записью результата.

Если задан `WEB_UI_TOKEN`, запросы должны передавать токен в заголовке `x-admin-token` или cookie `admin_token`. Не оставляйте пустой токен при публикации панели за пределы приватной сети.

## Первичная инициализация таблицы

```bash
docker compose run --rm app python -m app.cli init-sheets
docker compose run --rm app python -m app.cli test-google-access
docker compose run --rm app python -m app.cli inspect-sheet
```

`init-sheets` добавляет отсутствующие служебные колонки справа и создаёт лист `Требуется проверка`.

## Первичная обработка 2026 года

```bash
docker compose run --rm app python -m app.cli backfill-2026
```

Команда обрабатывает строки, где целевые поля пустые, и не перезаписывает уже заполненные значения.

## Регулярная работа

Контейнер `worker` запускает планировщик и обрабатывает задания из веб-панели. Интервал регулярной проверки задаётся `SCHEDULER_INTERVAL_MINUTES`. Для ручного запуска:

```bash
docker compose run --rm app python -m app.cli process-due
```

Диагностика одной закупки:

```bash
docker compose run --rm app python -m app.cli check-purchase 012345678901
docker compose run --rm app python -m app.cli check-purchase 012345678901 --write
```

## Лист `Требуется проверка`

Лист создаётся автоматически с колонками: номер закупки, причина, статус проверки, данные, даты создания и обновления. После ручной проверки поставьте статус `Проверка завершена` и выполните:

```bash
docker compose run --rm app python -m app.cli sync-review-sheet
```

## Резервное копирование PostgreSQL

```bash
docker compose exec postgres pg_dump -U procurement procurement > backup.sql
```

Восстановление:

```bash
docker compose exec -T postgres psql -U procurement procurement < backup.sql
```

## Обновление производственного календаря

В MVP календарь рабочих дней реализуется на уровне доменных тестов и может быть расширен отдельным справочником. После добавления справочника обновляйте его миграцией или отдельной CLI-командой.

## Ротация Google-ключа

1. Создайте новый ключ сервисного аккаунта.
2. Замените файл на VPS.
3. Проверьте права сервисного аккаунта на таблицу.
4. Перезапустите контейнер приложения.

```bash
docker compose restart app
docker compose run --rm app python -m app.cli test-google-access
```

## Диагностика

Логи пишутся в JSON и содержат номер закупки, этап, источник и идентификатор запуска, когда эти данные известны. Ошибка одной закупки не останавливает весь цикл.

Типы ошибок разделяются на временную недоступность источника, отсутствие документа, ошибку парсинга, недостаток данных, конфликт с ручным значением, ошибку Google Sheets и ошибку БД.

## Ограничения MVP

- Адаптер ЕИС использует публичные страницы и документы без обхода CAPTCHA, авторизации и ограничений.
- Если публичная структура не позволяет уверенно извлечь поле, закупка отправляется на ручную проверку.
- OCR для сканов не включён.
- Реестровый номер не подбирается автоматически по названию товара.
- Чекбоксы Google Sheets записываются как булевы значения; визуальное оформление можно расширить отдельным batchUpdate.

Подробные правила: [docs/business-rules.md](docs/business-rules.md). Схема образца: [docs/sheet-schema.md](docs/sheet-schema.md).
